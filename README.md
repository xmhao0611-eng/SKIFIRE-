# SKIFIRE

SKIFIRE 是一款面向连载小说、长篇小说和短篇故事的本地 AI 小说写作控制台。它把项目大纲、小说记忆卡、章节方向、正文生成、正文润色、正文库管理、长篇体检和 AI 资料库放在一个网页控制台里，帮助作者更稳定地推进长篇创作。

## 核心能力

- 管理作品名、题材、平台、更新频率、核心卖点和主题风格。
- 整理项目大纲，维护核心卖点、世界观、人物关系、阶段主线、伏笔和长期回收方向。
- 分析小说记忆卡，记录下一步、最大连载风险、未来章节建议和续写记忆。
- 推断后续章节方向，让正文生成先看大纲、记忆卡、最新章节和章节方向。
- 管理正文库，支持章节搜索、状态筛选、备注、保存、单章导出、批量导出和全本导出。
- 支持正文润色、精修润色、发布检查、读者反馈、断更恢复、季卷规划和完结规划。
- 维护 AI 资料库，用于长篇事实压缩、人物状态、世界观设定、伏笔线索和质量债务追踪。
- 提供使用统计，估算各功能的输入、输出和总 Token 消耗。
- 内置多套深色主题和明亮主题，并带有基础可读性检查。

## 推荐写作流程

1. 在写作首页填写作品基础信息并保存到项目库。
2. 把已有正文导入正文库，或在“已有小说/前文”里粘贴未完成内容。
3. 点击“整理大纲”，确认大纲后应用到项目。
4. 点击“分析记忆卡”，让软件总结当前剧情状态和后续风险。
5. 点击“推断章节方向”，生成接下来两章或四章的写作方向。
6. 点击“生成正文”，默认一次生成两章。
7. 检查质量门结果，必要时送去润色或发布检查。
8. 点击“保存章节”，把正文保存进正文库。

## 页面结构

- 写作首页：当前任务、AI 参考状态、五步写作流程、生成正文结果。
- 正文库：章节目录、正文编辑器、章节信息、TXT 导出。
- 使用统计：查看各功能的请求次数、输入 Token、输出 Token 和总 Token。
- AI 接口：管理本地 AI 引擎和第三方模型接口。
- 高级工具：正文润色、发布检查、读者反馈、断更恢复、季卷规划、完结规划和长篇体检。

## 主要文件

- `apps/serial-control/index.html`：网页控制台入口。
- `apps/serial-control/app.js`：前端交互和写作流程。
- `apps/serial-control/styles.css`：主题和界面样式。
- `scripts/serve_web_control.py`：本地网页服务和 AI 调用桥接。
- `scripts/start_serial_novel_app.cmd`：Windows 一键启动脚本。
- `scripts/install_or_update_plugin.cmd`：本机安装或更新脚本。
- `scripts/package_plugin.cmd`：生成便携包和一键安装包。
- `skills/novel-writing/SKILL.md`：小说写作流程说明。
- `assets/story-bible-template.md`：项目大纲模板。
- `使用说明.txt`：面向作者的完整使用说明。
- `SYNC.md`：跨电脑同步和迁移说明。

## 本地启动

双击：

```text
scripts\start_serial_novel_app.cmd
```

或者在 PowerShell 中运行：

```powershell
python scripts\serve_web_control.py --port 8787
```

然后打开：

```text
http://127.0.0.1:8787/
```

## 跨电脑使用

推荐使用 `scripts\package_plugin.cmd` 生成一键安装包。另一台电脑解压安装包后，双击安装脚本即可更新软件。小说项目默认保存在 `projects/`，运行缓存默认保存在 `.runtime/`，不会被安装包覆盖。

如果需要长期多电脑维护，可以把源码放在 GitHub 仓库里同步。不同电脑上的小说项目建议通过网页控制台的“导出项目”和“导入项目”迁移。

## 隐私说明

SKIFIRE 默认在本机运行。项目正文、设置、运行缓存和统计记录默认保存在本地电脑。发布源码时不应上传 `projects/`、`.runtime/`、`dist/`、安装包、运行日志或个人作品数据。

连载小说写作控制台 使用说明
版本：0.29.1

一、这个软件是做什么的

这是一个专门给连载小说和短篇故事使用的本地写作控制台。

它可以帮你：
1. 管理一本连载小说或短篇故事的基础信息。
2. 规划卷/季、项目大纲和章节方向。
3. 保存每一章正文。
4. 调用本机 Codex 直接生成内容。
5. 通过小白化 AI 接口对接 Codex 和其他模型，默认只看当前服务、当前模型和是否可用。
6. 通过侧栏主题入口打开色卡墙，切换 31 套配色主题，包括深色主题和明亮主题，并显示主色、背景色、文字色和可读性状态。
7. 在写作首页维护可编辑“项目大纲”，让后续章节始终沿着长期主线走。
8. 项目大纲固定为七个板块，并细分为核心卖点、世界观、人物、关系、阶段主线、伏笔和长期回收的小字段。
9. 大纲里的关键信息会要求标注【已确定】【推断】【待确认】，并尽量带来源和更新时间，方便后续更新时判断哪些能改、哪些不能改。
10. 整理大纲支持初次整理、更新大纲、检查大纲三种模式，检查结果可以进入项目大纲抽屉里的作者确认区，再根据确认优化大纲。
11. 在正文库搜索、筛选章节，查看最新章节标记，并导出 TXT，支持单章节、批量章节、全部章节。
12. 正文库改成三栏写作台：左侧章节目录、中间正文编辑器、右侧章节信息和快捷操作。
13. 正文库新增专注模式，写正文时可以隐藏章节目录和工具栏，只保留当前章节编辑区。
14. 正文库左侧章节列表改成紧凑卡片，右侧编辑区增加固定工具栏、字数统计和“已保存 / 未保存修改”状态。
15. 顶部保留“写作首页、正文库、使用统计、AI 接口”，其它功能统一放进“高级工具”卡片面板。
16. 在高级工具里润色章节正文，支持“普通润色”和“精修润色”两种方式。
17. 在高级工具里新增“完结规划”，可以设置作品阶段、预计剩余章数、结局类型、必须回收伏笔和必须交代人物。
18. 完结规划支持从正文库扫描线索、生成完结诊断、整理待回收伏笔、生成倒排完结大纲、检查是否可以完结。
19. 倒排完结大纲可以写入正文库，自动变成后续待写章节计划，已有正文的章节不会被覆盖。
20. 开启准备完结或完结中后，生成正文会额外参考完结规划和正文库待写计划，减少新坑，优先回收伏笔和推动最终结局。
21. 在写作首页直接查看正文库进度、小说记忆卡覆盖到第几章、章节方向对应哪几章。
22. 在左侧“创作模式”里切换连载小说和短篇故事；短篇模式会改成设定、骨架、正文、检查的四步流程。
23. 短篇模式新增“检查短篇”，专门检查故事闭环、主角动机、反转高潮、结尾余味、节奏废话和修改优先级，并给出“可保存 / 建议精修 / 结构需重改”状态。
24. 写作首页、编辑抽屉和正文库做了防卡顿优化，输入、搜索和章节切换会延迟或局部刷新，不再反复重算整份项目。
25. 启动脚本会先清理旧的后台服务，避免重复启动后网页打不开。
26. Codex 自动检测会优先使用本机可访问的 AppData 路径，避免误选 WindowsApps 拒绝访问路径。
27. 安装时会尝试生成桌面“启动小说控制台.exe”，双击即可清理旧服务、启动网页并打开浏览器。
28. AI 接口页新增“系统诊断”，可以查看网页服务、Codex、Python、项目库和启动器状态，并一键复制排查信息。
29. AI 接口支持“模型方案”和“功能绑定”：整理大纲、小说理解、章节方向、生成正文、正文润色可以分别指定使用哪个模型。
30. 预计发送字数改为停顿后估算，并缓存正文库来源文本，只有章节正文变化、新增或删除章节后才重建前文来源。
31. 短篇“送去精修”会把检查意见一并带入精修提示词，不只是带正文。
32. 写作首页改成任务工作台：先显示当前要写什么、AI 会参考什么、下一步该点哪里，再把手动步骤放在下面。
33. 写作首页新增“创作策略”：标准模式为未来4章建议、推断下两章；长线模式为未来8章趋势、推断下四章，但正文默认仍只写前两章，后两章只做牵引。
34. 新增“极简模式”：隐藏手动流程、生成设置、诊断和高级参数，只保留日常写作要看的任务、参考资料和下一步。
35. 检查大纲改为轻量检查模式，只发送必要的大纲、AI资料库/正文依据摘录和小说理解摘要，减少 Codex 因上下文过重而失败的概率。
36. 修复检查大纲后处理失败：确认项提取和 AI资料库小节提取不再共用同名函数，避免检查结果被错误覆盖成“生成大纲失败”。
37. 作者确认区升级为结构化确认项，分别显示“问题、AI建议、作者需要决定”，并在根据确认优化大纲时按结构化内容传给 AI。
38. 新增“使用统计”一级菜单，自动记录各功能输入、输出和总 Token，支持 24小时、7天、30天和全部筛选。
39. 使用统计页改成紧凑仪表盘，空状态会提示下一步操作，并支持把当前项目里已有内容导入为历史估算。
40. 修复网页端生成正文误触发旧技能检查的问题，生成会使用内置小说写作流程，不再依赖当前 Codex 会话是否加载写作技能。
41. 服务端也会清洗旧项目、旧页面、旧模型方案里残留的旧技能触发语，避免再次被 Codex 技能检查拦截。
42. AI 请求增加防卡住保护：如果网页等待异常过长，会自动停止等待并释放页面，不会自动降级模型。
43. 大纲作者确认区支持更多“需要确认”标题写法，确认项会自动展开，并区分待确认和已处理状态。
44. AI资料库新增章节内容 hash 追踪：旧章节正文、标题或备注修改后，会重新进入资料库更新队列。
45. 保存章节会自动标记资料库脏章节，避免旧事实继续影响大纲、记忆卡、章节方向和正文生成。
46. 正文库待写计划会优先成为下一章：生成、推断和保存正文会从最小的空计划章节继续，不再被后续待写计划推到末尾。
47. 生成正文保存时会优先写回同编号的空计划章，并保留/合并原章节备注，适合完结倒排计划和章节大纲落地。
48. 正文库“AI 输出写入本章”增加正文安全校验，避免把体检报告、检查文本或失败日志误写进章节正文。
49. 新增“长篇体检”，可以在高级工具里检查追读欲、剧情推进、人物动机、对话口吻、设定连续性、AI痕迹和 3 万/5 万字维护点。
50. 长篇体检结果可以一键“送去润色”，作为精修上下文处理当前章节。
51. 长篇体检会额外读取开头前三章参考，专门检查开篇抓人、三章契约和早期读者承诺。
52. 5 万字维护拆分为“结构体检”和“结构刷新”：运行体检只记录已检查，大纲/AI资料库/章节方向更新后才算刷新进度。
53. 生成正文增加安全保护：生成中不能重复点击，检查文本、状态提示和失败日志不能误保存进正文库。
54. 生成正文增加结果校验：AI 返回技能不可用、调试包装或 Codex 错误日志时，不再误判为正文。
55. 生成前会检查小说记忆卡和章节方向是否落后正文库，明显错位时会先暂停生成。
56. 正文库当前章节有未保存修改时，生成正文、长篇体检和保存项目库前会先静默保存。
57. 同一份生成正文保存后不会因为重复点击再次写入正文库，减少重复章节。
58. 正文生成会读取 AI资料库快速更新的新增资料，避免深度整理前漏掉新近章节事实。
59. 正文库编辑正文时会在停顿后自动保存草稿，同时保留手动保存按钮。
60. AI资料库的更新结果会单独保存，不再误覆盖项目大纲整理结果。
61. 旧项目里残留的“未来5章建议”在前端和服务端发送给 AI 前都会标记为历史参考，避免标准模式被旧流程带偏。
62. 项目库保存改成原子写入，并保留上一版 .bak 备份；断电导致主项目文件损坏时，会尽量从备份打开并提示你检查后再保存。
63. AI 生成会识别输出被截断、内容过滤、Codex 空输出和上下文窗口爆掉等情况，不会把这些失败内容当成正文保存。
64. 生成正文失败后会出现恢复面板，可以一键省量重试、单章重试、查看最近 Codex 日志或复制错误详情。
65. 生成正文后会自动显示保存前质量门，检查是否可保存、是否分章匹配、是否明显偏短、是否缺标题或疑似截断。
66. 保存前质量门可以把当前生成正文和质量门意见直接送去润色、发布检查，或打开长篇体检入口。
67. AI 长任务新增本地状态记录：刷新页面后也能看到上次 AI 调用是完成、失败还是可能被刷新中断。
68. 服务端会拦截旧版 $novel-writing 技能不可用提示，避免把技能失败文案当成正文生成成功。
69. 其他 OpenAI 兼容接口的后台等待时间会按功能类型调整，不再固定 90 秒，长大纲、长正文和体检更稳定。
70. 生成正文会读取 AI资料库里的人物状态库，减少长篇角色口吻、人物关系、伤势和秘密漂移。
71. 长篇正文库已有 6 章以上但 AI资料库为空时，会先暂停生成，要求快速更新 AI资料库。
72. AI资料库未读取章节超过 4 章缓冲时，会先暂停生成，避免正文基于过期事实继续写。
73. 生成正文保存入正文库时，会在章节备注里自动追加“本章兑现记录”，记录对应方向、必须出现内容、质量门状态和后续 AI资料库核对点。
74. AI资料库快速更新会读取章节备注里的“本章兑现记录”，生成“章节方向兑现核对”，让后续大纲、记忆卡和正文生成知道哪些计划已兑现、哪些需要补偿。
75. AI资料库面板会直接显示“章节兑现核对”摘要，方便快速查看哪些计划已兑现、哪些偏航、哪些伏笔需要补偿。
76. AI 调用有全局运行保护：一个 AI 任务未完成时不能启动第二个 AI 任务，避免大纲、资料库、体检和正文生成结果互相覆盖。
77. 保存前质量门会检查章节推进、读者奖励、章末压力、重复句、解释腔和 AI 套话风险，减少水文正文直接入库。
78. 写作首页新增修改任务清单，可以把保存前质量门、长篇体检和发布检查里的问题转成待处理任务，并标记已处理或忽略。
新增：修改任务清单新增“应用复查结论”，长篇体检或发布检查复查后，可以把已解决、不适用、仍存在等结论回填到任务状态。
79. 浏览器端行为自测钩子改为页面内 1 像素可点击按钮，自动验证能用真实点击触发自测流程。
80. 服务端会保存最近 AI 请求的结构化任务记录，只记录任务、模型、耗时、输入/输出长度、Token 用量和错误摘要，不保存 API Key、完整提示词或正文。
81. 服务端最近任务记录会优先读取模型返回的真实 usage；没有真实 usage 时才使用本地估算，并会自动合并到使用统计。
82. 生成正文会在 AI 调用入账前先校验返回内容；如果返回的是错误日志、技能失败提示或截断提示，会按失败任务统计，并保留输出消耗。
83. 保存前质量门会检查“必须出现/读者奖励”和章节方向关键词是否兑现；未明显命中时给出警告，避免正文偏离原计划。
84. 浏览器本地缓存损坏时会显示恢复提示，提醒先从项目库或导出备份恢复，避免误以为空项目覆盖原项目。
85. 导入坏备份或错误 JSON 文件时不会覆盖当前项目，并会明确提示失败原因。
86. 生成正文保存入正文库后，会等待项目库同步结果；同步失败时会提示“浏览器已保存、项目库未同步”。
87. 生成正文在“必须出现/读者奖励”未填写时，会从章节方向里自动提取核心目标、读者奖励和章末钩子作为质量锚点。
88. 小说记忆卡新增硬规则：未来章节建议只能放在未来建议板块，续写记忆卡只记录已发生事实、当前状态和未回收债务。
89. AI 任务状态栏会显示上次发送前快照是否写入项目库，方便断电或刷新后判断数据有没有保存住。
90. AI 长任务发送前会先保存浏览器状态，并静默写入项目库；刷新、断电或浏览器崩溃后，最新输入和章节编辑内容更容易恢复。
91. 浏览器端行为自测改为页面内隐藏钩子执行，验证工具可以点击自测按钮并读取结果，确认真实页面运行时没有回归。
92. 新增浏览器端行为自测，可以验证错误输出拦截、两章正文拆分、计划章优先续写和 AI资料库待扫描规则。
93. AI资料库增量扫描会重新读取缺少章节 hash 记录的已写章节，旧项目升级后不会误判资料库已经完整覆盖。
94. 整理大纲生成结果不会提前算作结构刷新完成，必须应用整理结果写入项目大纲后才记录本次结构刷新。
95. 保存前质量门升级为二次确认：生成正文存在警告时，第一次点击保存只提示问题，必须点“确认仍保存”才会写入正文库。
96. 章节方向新增生命周期提示，可以区分“可使用、仍可用、已用完、需重推”，保存章节后首页推荐会随正文库进度更新。
97. AI资料库和最近章节参考的长章节摘录改为“开头 + 中段采样 + 结尾”，减少中段伏笔、道具来源和人物变化漏读。
98. 长篇项目生成前会检查 AI资料库状态，正文库已有较多章节但资料库为空时，会提示先快速更新。
99. 生成前会检查输出预算，两章模式或短篇目标过高但最大输出偏低时，会先提示改成单章或提高最大输出。
100. 生成正文保存入正文库后，会自动同步到项目库，减少断电或浏览器缓存丢失导致章节回退的风险。
101. 验证脚本会检查重复 HTML id、重复前端函数名和关键 AI 行为，减少后续迭代时把稳定流程改坏的风险。
102. 导出和导入项目，方便备份或换电脑使用。
103. 打开项目或导入项目前会检查 AI 是否运行、当前章节是否未保存、生成正文是否未入库，以及浏览器缓存是否刚恢复异常，避免误替换当前写作现场。
104. 保存前质量门会检查对话/人物交锋和场景物件/感官细节，提示说明书式正文、现场感不足等问题。
105. 保存前质量门的现场感识别会避开“意味着”等普通词里的单字误报，减少质量门假通过。
106. 保存前质量门会提示“好的，以下是正文”等 AI 前言包装，避免说明文字混进章节正文。
107. 生成正文和推断章节方向会加入“流程循环防护”，要求规则、系统、复核、档案等信息必须落到人物选择、身体代价、场景变化或读者奖励。
108. 保存前质量门会检查规则术语密度、系统提示式章末钩子和连续重复句，避免流程报告腔或误复制正文直接入库。
109. 项目摘要和生成前检查会优先按正文库最新章节续写；如果旧 currentChapter 字段落后正文库，会明确提示但不会自动改动你的项目数据。
110. 保存前质量门加入修改任务时，会把“规则术语过密、系统式章末、连续重复句”等提示转成具体改写任务，方便直接照着修。
111. 长篇体检和发布检查如果按“问题、影响、怎么改、优先级、验证”输出，软件会自动提取成更完整的修改任务，方便形成检查、修改、复查闭环。
112. 未处理修改任务会进入后续 AI 上下文，正文生成、正文润色、长篇体检和发布检查都会参考这些待办，减少同类问题反复出现。
113. 长篇体检和发布检查会复查未处理修改任务，逐条判断已解决、仍存在、部分解决或不适用；仍存在的问题会继续进入结构化修改任务。
114. 保存前质量门会提示当前生成范围仍未处理的高优先级修改任务；如果你确认正文已经修复或任务不适用，需要再点一次“确认仍保存”才会入库。
115. 本章兑现记录会单独写入未处理高优先级修改任务，后续 AI资料库更新时能看到哪些保存章节仍需要补偿或复查。
116. AI资料库面板会把“保存记录待复查”直接显示在章节兑现核对和风险区，方便快速发现哪些章节曾带高优先级待办入库。
117. AI资料库新增“质量债务”卡片，会汇总当前未处理修改任务和保存记录待复查数量，生成正文前更容易先处理质量风险。
118. 写作首页新增“质量债务”状态卡，生成正文按钮也会在有未处理风险时提示“带风险避坑”。
119. 修改任务清单新增“AI处理任务”：勾选 3-5 条待办后，可以让 AI 分类任务、生成修改方案、按任务修正文、复查任务是否解决；AI修正文不会自动覆盖章节，需要你点“应用到当前章节”后再保存。

2. 左侧先选择“创作模式”：连载小说或短篇故事。

四、最简单写作流程

新版默认打开就是“写作首页”，现在它先显示“当前写作任务”和一个最大的“下一步”按钮。
你可以只看这个主按钮完成流程；需要细调时，再看下面的手动步骤。

如果你已经有一篇没有写完的小说，推荐这样用：

1. 先看页面上方的“当前写作任务”，确认现在要写的是哪一章。
2. 看右侧“AI 当前读取依据”，确认正文库最新章节是否正确。
3. 按最大的“下一步”按钮执行；也可以把已经写好的正文粘到“已有小说/前文”，或点击“从正文库汇总”。
4. 如果需要手动控制，再点击下方步骤里的“整理大纲”，先得到大纲整理结果。
5. 点击“项目大纲”摘要卡进入编辑抽屉，确认“大纲整理结果”，没问题后点“应用整理结果”。如果是检查大纲，先在项目大纲抽屉里的“作者确认区”处理确认项，再点“根据确认优化大纲”。
6. 点击“分析记忆卡”，让控制台输出下一步、最大风险、未来4章建议或未来8章趋势、续写记忆卡。
7. 点击“推断章节方向”，控制台会直接读取正文库最新章节结尾；标准模式推断下两章，长线模式推断下四章。
8. 点击“生成正文”。默认会一次写两章，两章中间用“===下一章===”分隔。
9. 满意后点击“保存章节”，控制台会自动拆成两个新章节保存到正文库。

如果你已经把章节保存在“正文库”里：

1. 先看页面上方的“AI 当前读取依据”，确认最新章节显示正确。
2. 点击“从正文库汇总”。
3. 在“生成设置”里选择“大纲模式”，点击“整理大纲”，或点击“项目大纲”摘要卡修改项目大纲。
4. 如果使用“初次整理”或“更新大纲”，点击“项目大纲”摘要卡，确认“大纲整理结果”后点“应用整理结果”。
5. 如果使用“检查大纲”，先点“提取确认项”，在“作者确认区”选择采用 AI 建议、保留原设定或按你的说明处理，再点“根据确认优化大纲”。
6. 点击“分析记忆卡”。
7. 点击“推断章节方向”，系统会直接读取正文库最新章节结尾。
8. 点击“生成正文”。
9. 如果是两章正文，直接点击“保存章节”，不用手动拆分。

可选填写：

1. 三张摘要卡：项目大纲、小说记忆卡、章节方向。首页只显示摘要，点击卡片进入编辑抽屉。
2. 生成设置：默认收起来，里面可以调整大纲模式、省量模式、最近前文字数，也可以把“生成正文范围”从“一次写两章”改成“只写下一章”。
3. 作者确认区在“项目大纲”编辑抽屉里；可选要求收在“生成设置”里，避免写作首页长期堆满输入框。
4. 正文润色：在高级工具里粘贴单章正文，选择普通润色或精修润色，再点击“开始润色”。

如果你只是想快速写正文，其他页面都可以先不用管。

短篇故事模式：

1. 左侧“创作模式”选择“短篇故事”。
2. 填写作品名、类型、目标字数和核心看点。
3. 如果已有草稿，把草稿粘到“已有小说/前文”；如果没有草稿，也可以只填设定。
4. 点击“整理设定”，得到短篇设定/骨架。
   短篇设定同样按七个板块整理：题材风格、主角和处境、核心冲突、故事骨架、关键反转、高潮与结尾、情绪落点。
5. 点击“生成骨架”，系统会自动理解已有草稿，并输出开场钩子、主角困境、隐藏真相、高潮选择、结尾落点等完整骨架。
6. 点击“生成短篇”，完整短篇会出现在正文框里。
7. 点击“检查短篇”，检查故事闭环、主角动机、反转高潮、结尾余味和节奏问题，并查看状态：可保存、建议精修、结构需重改。
8. 需要修改语言时，点检查区里的“送去精修”；检查意见会一起进入精修提示词。确认可以保留时，点正文框右上角“保存短篇”放入作品库。

高级工具默认不占顶部导航。
需要时点击右上角“高级工具”，再从卡片进入正文润色、发布检查、读者反馈、断更恢复、季/卷规划或完结规划。

长篇体检怎么用：

1. 正文库累计到 3 万字后，建议进“高级工具”点“长篇体检”。
2. 体检会重点看追读欲、章节推进、人物动机、对话口吻、设定连续性和 AI 痕迹。
3. 正文库累计到 5 万字后，体检会额外提醒是否需要刷新大纲、正文规划、人物状态、世界观、伏笔、时间线和道具体系。
4. 体检结果只给修改任务和风险判断，不会直接改正文。
5. 如果要修改当前章节，点“送去润色”，软件会把体检报告带入正文润色的精修上下文。
6. 到 5 万字后，体检只代表“已检查”；仍建议继续更新大纲、深度整理 AI资料库并重推章节方向，完成真正结构刷新。

关于省量模式：

写作首页默认开启“省量模式”，入口收在“生成设置”里。
开启后，控制台会尽量减少 Codex 调用时发送的前文字数。

推荐流程：

1. 第一次粘贴小说后，先选择“初次整理”，点击“整理大纲”。
2. 控制台会生成“大纲整理结果”，只整理故事骨架，不生成未来章节建议。
   新版大纲会按七个固定板块输出，每个板块带固定小字段。关键信息会尽量带【已确定】【推断】【待确认】、来源和更新时间。
3. 确认结果没问题后，点“应用整理结果”，这样后续生成才会使用新版大纲。
4. 后续有新章节时可以选择“更新大纲”，它会尽量保留你手动写过的设定，只补充变化。
5. 如果不想马上改大纲，只想查问题，选择“检查大纲”，它只输出冲突、缺失和需要确认。
6. 检查后可以点“提取确认项”，逐条选择处理方式，也可以在“作者补充确认”里写你的拍板意见。
7. 点“根据确认优化大纲”后，系统会自动切到“更新大纲”，把检查报告和作者确认一起交给 AI，生成新版“大纲整理结果”。
8. 确认新版结果没问题后，再点“应用整理结果”。检查结果不会直接覆盖当前大纲。
9. 再点击“分析记忆卡”，生成“小说理解结果”，负责下一步、风险、未来4章建议或未来8章趋势、续写记忆卡。作者需要确认的问题放在项目大纲里处理，不放进记忆卡。
10. 后面“推断章节方向”和“生成正文”会优先使用“项目大纲 + 小说记忆卡/小说理解结果 + 正文库最新章节结尾 + 章节方向”。
11. “推断章节方向”会先分析项目大纲，再分析小说理解结果，再读取正文库最新章节结尾；标准模式输出两个章节的完整推断，长线模式输出四章方向，其中前两章详细、后两章只做牵引。
12. “生成正文”的固定顺序是：先看项目大纲，再看小说记忆卡/小说理解结果，再看正文库最新章节结尾，再看章节方向，最后直接生成正文。
13. “生成正文”默认只按前两章推断写正文，保存时自动拆章；长线模式里的第三、第四章只用于伏笔和节奏牵引，不会提前写正文。想更稳或只写一章，可以在“生成设置”里切换。
14. 同时只带最近一小段前文，默认 5000 字。

如果你觉得 AI 忘了细节，可以把“最近前文字数”调到 8000 或 12000。
如果你想最省，就选 3000。

页面会显示“预计发送字数”，用来判断本次调用大不大。

高级功能怎么用：

每个功能标题旁边都有“操作说明”。
点开后会显示 3 步左右的使用方法。
不确定怎么填时，先点“操作说明”，按步骤填完再点该功能的生成按钮。

六、怎么用 Codex 直接生成

这是最推荐的小白模式。

1. 点击顶部的“AI 接口”。
2. 服务商保持为“Codex 本机一键，推荐”。
3. Codex 命令/路径保持为 auto。
4. 点击“检测 Codex”。
5. 如果显示 Codex 可用，就说明本机可以调用。
6. 回到“写作首页”或“正文库”。
7. 写作首页可以直接点击“生成正文”；高级页面可以点击对应的“生成提示词”按钮。
8. 再回到“AI 接口”，点击“直接生成”。
9. 等待几十秒到两分钟，AI 输出会出现在下方。

注意：
Codex 模式不需要填写 API Key。
只要你的电脑已经登录并安装 Codex，就可以使用。

关于温度和最大输出：

默认会开启“智能分配参数”。
开启后不需要你手动调温度和最大输出，控制台会按模块自动分配：

1. 整理大纲：更稳，输出中等。
2. 分析记忆卡：更稳，输出较长。
3. 推断章节方向：兼顾推理和创意，输出较长。
4. 生成正文：创作空间更大，输出最长。
5. 正文润色：普通润色偏纠错顺句；精修润色会在不重写剧情的前提下加强语言质感、节奏、画面和情绪。

如果你关闭“智能分配参数”，才会使用“高级参数”里的手动温度和最大输出。

AI 接口里的“模型方案”负责决定用哪个服务和模型。
例如你可以让“整理大纲”和“小说理解”使用稳定分析模型，让“生成正文”使用正文长文模型，让“正文润色”使用润色精修模型。
生成或测试之后，“上次调用”会显示实际使用的服务、模型、功能和最大输出。

关于 Token 使用统计：

1. 顶部“使用统计”在“正文库”和“AI 接口”之间。
2. 每次 AI 调用会自动记录输入 Token、输出 Token、总 Token、功能、服务、模型和耗时。
3. 如果接口返回 usage，页面会显示“实际”；如果没有返回 usage，页面会按文本长度本地估算。
4. Codex 本机调用通常没有精确 usage，所以主要显示估算值。
5. 统计只保存在当前电脑当前浏览器里，可以导出 JSON，也可以清空，不影响小说项目。
6. 点击“导入历史估算”可以扫描当前项目里已有的大纲、记忆卡、章节方向、正文库正文、润色结果和完结规划结果，生成一批估算记录。历史估算不是精确账单，只用于先看大概消耗分布。

补充：
其他 HTTP API 会直接使用这些温度和最大输出参数。
Codex 本机模式会把当前模块类型和推荐参数写入任务提示里，让 Codex 知道这是整理大纲、分析记忆卡、推断章节方向、生成正文还是正文润色。


七、怎么润色正文

1. 打开“写作首页”。
2. 点击右上角“高级工具”。
3. 点击卡片“正文润色”。
4. 在“润色方式”里选择：
   - 普通润色：纠错、顺句、适度增强。
   - 精修润色：语言质感、叙事节奏、画面和情绪全面提升，但不重写剧情。
5. 把需要润色的一章粘到“原章节”。
6. 点击“开始润色”。
7. 润色完成后，可以点击“复制润色结果”，也可以点“放入生成正文”再保存。

如果刚刚已经生成了正文，也可以点“从生成正文带入”，不用再复制一遍。


八、怎么保存正文

1. 点击顶部“正文库”。
2. 点击“新建章节”。
3. 填写章节号、状态、标题。
4. 在“正文”里写内容，或者把 AI 输出写入这里。
5. 点击“保存章节”。

如果 AI 输出已经在“AI 接口”里：

1. 点击“正文库”。
2. 选择当前章节。
3. 点击“AI 输出写入当前章节”。
4. 点击“保存章节”。

如果要导出 TXT：

1. 点击“正文库”。
2. 在左侧可以用“搜索章节”和“状态筛选”快速找章节，带“最新”标记的是正文库最新有效章节。
3. 打开右上角“导出 TXT”菜单。
4. 单章导出：选择当前章节后，点击“单章”。
5. 批量导出：在“批量选择章节”里选多个章节，点击“批量”。
6. 全本导出：点击“全本”，会把所有有正文的章节合成一个 TXT。

如果还想保留 Markdown 备份，可以点击“导出当前 Markdown”。


九、怎么管理连载进度

顶部只保留三个常用页面：

1. 写作首页
整理大纲、分析记忆卡、推断章节方向、生成正文、保存章节。

2. 正文库
保存章节正文。
点“操作说明”可以看到怎么保存、导出和写入 AI 输出。

3. AI 接口
设置 Codex 或其它模型接口。

其它功能在右上角“高级工具”里：

1. 正文润色
普通润色或精修润色单章正文。

2. 发布检查
发章节前检查读者奖励、章节推进、钩子、连续性、伏笔债务。

3. 读者反馈
粘贴评论或数据，让 AI 帮你分析哪些反馈值得吸收。

4. 断更恢复
如果停更一段时间，用它整理重启方案。

5. 季/卷规划
规划一卷或一季的主线、跨度、前几章钩子。

6. 完结规划
当你准备收尾时使用。建议流程：
先把“作品阶段”改成“准备完结”或“完结中”，填写预计剩余章数、结局类型、必须回收伏笔、必须交代人物和作者最终决定。
建议先点击“从正文库扫描线索”，让 AI 根据正文库章节索引、最近章节和备注整理未回收线索。
然后依次点击“生成完结诊断”“整理待回收伏笔”“生成倒排完结大纲”“检查是否可以完结”。
倒排大纲确认后，可以点击“倒排计划写入正文库”，软件会把后续章节写成正文库里的待写计划。已有正文的章节不会被覆盖。
开启准备完结或完结中后，写作首页会显示完结进度，生成正文时也会参考完结规划和正文库待写计划，优先收束主线、回收伏笔和靠近最终结局。


十、怎么接入其他 AI 模型

如果不用 Codex，而是用其他平台：

1. 点击“AI 接口”。
2. 打开“高级参数”。
3. 在“服务商”里选择平台。
4. 粘贴该平台的 API Key。
5. 模型和接口地址通常会自动填好。
6. 在“模型方案”里保存当前方案。
7. 在“功能使用哪个模型”里选择哪些功能使用这个方案。
8. 点击“测试接口”。
9. 测试成功后点击“直接生成”。

如果你是小白，优先使用 Codex 本机一键模式。
Codex 不需要 API Key，最省事。


十一、怎么保存、导出、导入项目

保存到本机项目库：

点击左侧“保存到项目库”。

打开以前的项目：

1. 点击“刷新项目库”。
2. 在“项目库”下拉框选择项目。
3. 点击“打开选中项目”。

导出项目：

点击左侧“导出”，会导出一个 json 文件。

导入项目：

点击左侧“导入”，选择之前导出的 json 文件。


十二、换电脑怎么用

最简单的方法：使用完整一键安装包

1. 在主电脑打开插件目录。
2. 双击：
   scripts\package_plugin.cmd
3. 在 dist 文件夹里找到：
   novel-writer-installer-0.29.1.zip
4. 把这个 installer zip 发到另一台电脑。
5. 在新电脑解压这个 installer zip。
6. 双击：
   一键解包安装.cmd
7. 等安装完成后，程序会尝试自动启动网页端。
8. 如果没有自动打开浏览器，就手动打开：
   http://127.0.0.1:8787/

小说项目建议用网页端“导出项目”和“导入项目”迁移。

备用方法：手动安装普通插件包

如果你拿到的是 novel-writer-0.29.1.zip，而不是 installer 安装包：

1. 解压 novel-writer-0.29.1.zip。
2. 进入解压后的 novel-writer 文件夹。
3. 双击：
   scripts\install_or_update_plugin.cmd
4. 再双击：
   scripts\start_serial_novel_app.cmd
5. 打开：
   http://127.0.0.1:8787/


十三、常见问题

1. 网页打不开怎么办？

先确认是否已经启动：

C:\Users\Administrator\plugins\novel-writer\scripts\start_serial_novel_app.cmd

如果还是打不开，检查浏览器地址是不是：

http://127.0.0.1:8787/


2. 点“直接生成”很慢怎么办？

Codex 本机生成通常需要几十秒到两分钟。
生成长内容时更慢，这是正常的。


3. Codex 检测成功，但生成失败怎么办？

先点击“检测 Codex”确认可用。
如果可用但生成失败，把“AI 输出”里的错误信息发给 Codex，让它继续修复。


4. API Key 要不要保存？

如果是自己的电脑，可以勾选“把 API Key 保存到本机浏览器 localStorage 里”。
如果是公共电脑，不建议保存。

Codex 模式不需要 API Key。


5. 项目会不会被更新覆盖？

正常不会。
安装/更新脚本会保留 projects 文件夹。
但是重要小说仍建议经常导出备份。


十四、推荐日常写作流程

极简流程：

1. 打开网页端。
2. 在“已有小说/前文”粘贴没写完的小说。
3. 点击“整理大纲”，再点“项目大纲”摘要卡进入编辑抽屉，确认“大纲整理结果”后点“应用整理结果”。
4. 点击“分析记忆卡”。
5. 点击“推断章节方向”，控制台会读取正文库最新章节结尾。
6. 点击“生成正文”，默认一次写出两章。
7. 点击“保存章节”，系统会按分隔符自动拆章入库。

进阶流程：

1. 在“总览”更新当前项目状态。
2. 回到“写作首页”，点击对应摘要卡修改项目大纲和小说理解结果。
3. 点击“推断章节方向”，控制台会读取正文库最新章节结尾。
4. 点击“生成正文”，默认一次生成两章。
5. 如需优化文字，打开高级工具里的“正文润色”处理单章。
6. 到“正文库”保存正文或导出 TXT。
7. 到“发布检查”检查是否可以发布。
8. 发布后更新项目状态和正文库。


十五、最简单一句话用法

打开网页，先确认“AI 当前读取依据”里的最新章节，然后按写作首页上方 5 个按钮从左到右执行：整理大纲、分析记忆卡、推断章节方向、生成正文、保存章节。控制台会自动读取正文库最新章节结尾，默认一次生成并保存两章。


十六、AI资料库更新卡住怎么办

如果刷新网页后点击“快速更新 AI资料库”没有反应，先看左侧项目库是否已经打开当前作品。
新版会在只有一个项目且页面安全空白时自动打开；如果检测到未保存正文或临时内容，会提示你先点击“打开选中项目”，避免空项目覆盖旧数据。

AI资料库快速更新现在按小批次执行：每批最多读取 2 章，每章只截取关键内容。若 Codex 连接中断，已完成的批次会保留；下次再点“快速更新 AI资料库”会从未完成章节继续。

如果日志里出现 websocket 断开但随后 fallback HTTP 已经产出正文，软件会接收可用结果，不再把完整资料卡误判为失败。

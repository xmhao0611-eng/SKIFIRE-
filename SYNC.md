# 跨电脑同步和迁移说明

SKIFIRE 的文件分为两类：

- 应用源码：网页端、脚本、模板、样式和写作流程说明，可以跨电脑同步和更新。
- 小说数据：默认保存在 `projects/`，属于个人作品数据，不建议上传到公开仓库。
- 运行缓存：默认保存在 `.runtime/`，用于本机运行状态和临时记录，不需要同步。

## 最简单方式：ZIP 一键安装包

适合不想配置 GitHub 的情况。

1. 在主电脑双击 `scripts\package_plugin.cmd`。
2. 到 `dist\` 里找到生成的 `novel-writer-installer-版本号.zip`。
3. 把 installer zip 发到另一台电脑。
4. 在另一台电脑解压 installer zip。
5. 双击 `一键解包安装.cmd`。
6. 安装完成后，网页控制台会尝试自动启动。

一键安装包默认不会删除另一台电脑上的 `projects/` 小说项目。

## 推荐方式：GitHub 同步源码

适合以后经常在多台电脑上更新这个软件。

1. 在每台电脑安装 Git 或 GitHub Desktop。
2. 把源码仓库克隆到本地。
3. 修改前先拉取最新版本。
4. 修改后运行本地验证脚本。
5. 验证通过后提交并推送。
6. 另一台电脑拉取最新版本后，双击 `scripts\install_or_update_plugin.cmd` 更新本机应用。

## 新电脑首次使用

1. 安装 Python 3。
2. 获取 SKIFIRE 源码，可以用一键安装包，也可以用 GitHub。
3. 双击 `scripts\install_or_update_plugin.cmd`。
4. 双击 `scripts\start_serial_novel_app.cmd`。
5. 浏览器打开 `http://127.0.0.1:8787/`。
6. 进入 `AI 接口`，按页面提示检测本地 AI 引擎或填写第三方模型接口。

## 小说项目迁移

网页控制台保存的项目默认在 `projects/` 里。公开同步源码时不建议同步这个目录，避免个人作品泄露或互相覆盖。

迁移小说项目推荐两种方式：

- 在网页控制台使用“导出项目”，到另一台电脑再“导入项目”。
- 手动备份 `projects/` 文件夹，只在自己的电脑之间复制。

API Key 不会写进发布包。不同电脑需要各自填写；本地 AI 引擎模式通常不需要在网页里填写 API Key。
